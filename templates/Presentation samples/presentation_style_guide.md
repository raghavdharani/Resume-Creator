# Presentation Sample Style Guide

Reference sample:

- `Aatmika_Natarajan_CV_upd.pdf`

Use this folder as the global presentation reference library. Before exporting final resumes, the Presentation And Publishing Reviewer should inspect the files in this folder and apply relevant visual patterns unless the active user profile specifies a different style.

## Patterns To Reuse

- Compact, professional typography similar to Calibri.
- Restrained blue accent color for section labels and role metadata.
- Thin top blue rule.
- Pale blue header band.
- Large blue candidate name on the left side of the header.
- Right-aligned contact, profile, and title block on the right side of the header.
- Header title line should not wrap awkwardly. Shorten the title line before shrinking the font too much.
- Uppercase section labels with clear spacing.
- Candidate name, section labels, underline rules, company names, and body content should share the same master left gutter.
- In sample-inspired layouts, the left edge of the candidate name/header content defines the master gutter for the rest of the resume.
- Clean left alignment and predictable vertical rhythm.
- Narrow but readable page margins.
- Clear separation between summary, skills, work experience, education, and tools.
- Blue role/company/date metadata lines to create scannable hierarchy.
- Experience header rows should distribute company, role/title, and tenure across one line: company flush with the master gutter, role/title centered or visually balanced in the middle, dates right.
- Add a subtle vertical gap before each new company experience block.
- Tight bullet indentation with readable line wrapping.
- Selective bolding inside bullets only when it improves scanability.
- Strong page economy without making the page feel cramped.
- Core Strengths should be grouped into category rows, for example `Product & Delivery`, `AI & Technical`, and `Domain & UX`.
- Prefer `Category: skill | skill | skill` rows over long keyword strings or repeated-prefix two-column bullets.

## Patterns To Use Carefully

- Multi-column skills can be visually efficient, but may reduce ATS parsing. Use grouped text lines instead when ATS safety matters.
- Symbol bullets may render well in the sample but can render as question marks in generated PDFs. Verify output rendering or use hyphen bullets.
- Use hyphen bullets in generated PDFs unless a symbol bullet is verified to render cleanly.
- Dense experience sections are acceptable only when the content is highly relevant and readable.

## Presentation QA Checklist

- Candidate name, section label, underline rule, company name, and body content align to the same master gutter.
- Core Strengths is grouped by category and readable in under 10 seconds.
- Header has no awkward title wrapping.
- Company starts at the same left gutter as the candidate name and section label, while role/title and dates are spaced across the full experience header row.
- There is a subtle gap between different employer sections.
- PDF is two pages when the role and candidate seniority call for a two-page resume.
- PDF has zero question-mark bullet rendering issues.
- The document looks polished but not over-designed.

## Do Not Copy

- Candidate names
- Employers
- Career facts
- Metrics
- Bullets
- Technical claims
- Any private candidate content

Use samples for presentation inspiration only.
